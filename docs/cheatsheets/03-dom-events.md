# Cheat Sheet 03 — DOM & Events

> Reference for Phase 3 of the Joy Curry & Tandoor build.

---

## Selecting Elements

**`getElementById`** — Fastest; use when you know the exact ID.
```js
const menuRoot    = document.getElementById('menu-root');
const cartBadge   = document.getElementById('cart-badge');
const searchInput = document.getElementById('search-input');
```

**`querySelector`** — CSS selector syntax; returns the first match.
```js
const overlay = document.querySelector('.cart-overlay');
const active  = document.querySelector('.filter-btn--active');
```

**`querySelectorAll`** — Returns a NodeList (array-like) of all matches.
```js
const filterBtns = document.querySelectorAll('.filter-btn');
filterBtns.forEach(btn => btn.addEventListener('click', handleFilter));
```

## Rendering HTML

**`innerHTML`** — Set the entire HTML of a container. Used by all Joy Curry components.

```js
function renderMenu(items) {
  const html = items.map(item => MenuCard(item)).join('');
  document.getElementById('menu-root').innerHTML = html || ZeroResults();
}
```

> **Security rule:** never interpolate raw user input into `innerHTML`. Sanitise or use `textContent` for user-supplied strings.

**`textContent`** — Set plain text; safe from XSS.
```js
document.getElementById('cart-badge').textContent = cartState.totalItems;
```

## Event Listeners

**`addEventListener`** — Attach a handler to a specific element.
```js
document.getElementById('search-input').addEventListener('input', (e) => {
  filterMenu(e.target.value.trim());
});
```

**Event object properties**

| Property | What it gives you |
|----------|-------------------|
| `e.target` | The exact element that was clicked/changed |
| `e.currentTarget` | The element the listener is attached to |
| `e.target.value` | Current input value |
| `e.target.dataset.id` | `data-id` attribute as a string |
| `e.preventDefault()` | Stop form submit / link navigation |
| `e.stopPropagation()` | Stop event from bubbling up |

## Event Delegation

**Attach one listener to a parent instead of many listeners on each child.** Required for dynamically rendered content (cards generated by `innerHTML`).

```js
document.getElementById('menu-root').addEventListener('click', (e) => {
  const addBtn = e.target.closest('[data-action="add-to-cart"]');
  if (!addBtn) return;

  const itemId = Number(addBtn.dataset.id);
  cartState.addItem(itemId);
});
```

`closest()` walks up the DOM tree from the clicked element, returning the first ancestor that matches the selector — so clicking anywhere inside a card still resolves the correct target.

## Data Attributes

**`data-*`** — Embed structured data directly in HTML; read it from event handlers without querying the store.

```html
<button class="menu-card__add-btn"
        data-action="add-to-cart"
        data-id="42"
        aria-label="Add Lamb Rogan Josh to cart">
  Add
</button>
```

```js
const id     = Number(btn.dataset.id);       // "42" → 42
const action = btn.dataset.action;           // "add-to-cart"
```

## UI States

**Loading → Content → Empty → Error** — All four states must be handled in every list view.

```js
function renderMenu(state) {
  const el = document.getElementById('menu-root');
  if (state === 'loading') { el.innerHTML = SkeletonGrid();  return; }
  if (state === 'error')   { el.innerHTML = ErrorBanner();   return; }
  if (!items.length)       { el.innerHTML = ZeroResults();   return; }
  el.innerHTML = items.map(MenuCard).join('');
}
```

**Toggling a CSS class** — The canonical way to open/close drawers, modals, overlays.
```js
// Open
cartDrawer.classList.add('cart-drawer--open');
overlay.classList.add('cart-overlay--visible');

// Close
cartDrawer.classList.remove('cart-drawer--open');

// Toggle
filterBtn.classList.toggle('filter-btn--active');

// Conditional add/remove
cartBadge.classList.toggle('cart-badge--hidden', count === 0);
```

## ARIA for Accessibility

Announce dynamic changes to screen readers without a page reload.

```html
<span id="cart-badge" aria-live="polite">0</span>
<div  id="menu-root"  aria-live="polite"></div>
```

```html
<!-- Modal -->
<dialog class="options-modal" role="dialog" aria-modal="true" aria-labelledby="modal-title">
  <h2 id="modal-title">Customise your order</h2>
</dialog>
```

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| `document.querySelectorAll('.card').forEach(el => el.addEventListener('click', fn))` inside a render loop | Use event delegation on the container instead — dynamic children lose listeners on re-render |
| `e.target.dataset.id` used directly as a number | Always `Number(e.target.dataset.id)` — dataset values are strings |
| Calling `renderMenu()` before the DOM is loaded | Wrap in `DOMContentLoaded` or put script at bottom of `<body>` |
| Interpolating `userInput` directly into `innerHTML` | Use `textContent` or sanitise; raw `innerHTML` with user data is XSS |
| Forgetting `aria-live` on dynamically updated regions | Add it — screen readers won't announce `innerHTML` changes without it |
