Common Mistakes
pwd
Where am I right now?
/Users/samsid/dev/joy-curry-tandoor
🍛 Run this anytime you're lost — confirms you're inside the project
ls
Show everything in this folder
README.md backend/ docs/ frontend/ .gitignore
🍛 Use after creating files to confirm they actually appeared
cd
Move into a folder (change directory)
cd frontend/js ← go into a folder cd .. ← go up one level cd ~/dev/joy-curry-tandoor ← jump from anywhere
🍛 `cd frontend/js` to reach your JavaScript files
mkdir
Create a new folder
mkdir -p backend/routes/v1 ← -p creates parents too
🍛 Used once in setup to build the full project structure
touch
Create an empty file
touch frontend/js/menu-data.js
🍛 Creates menu-data.js before your first menu item
Ctrl+C
Stop whatever is currently running
(no typing — just hold Ctrl and press C)
🍛 Essential in Phase 4 to stop your running Node.js server
Cmd+K
Clear the terminal screen (nothing is deleted)
(keyboard shortcut inside Terminal or VS Code terminal)
🍎 Use when the terminal looks cluttered — it just scrolls clean
git clone
Download a repo from GitHub and link them
git clone https://github.com/SayemSiddique/joy-curry-tandoor.git
🍛 Used once in Phase 0 to get the repo onto your Mac
git status
Show what changed since the last save point
orange = modified red = new green = staged (ready)
🍛 Run this constantly. Before every commit, no exceptions.
git add
Stage files — include them in the next save point
git add . ← stage everything git add frontend/js/menu-data.js ← one file only
🍛 After writing menu items, `git add frontend/js/menu-data.js`
git commit
Create the save point with a message
git commit -m "feat: add chicken entrees to menu-data.js (M1.1)"
🍛 Every milestone gets one commit. Message = what you did + why.
git push
Upload save points to GitHub
git push origin main
🍛 Run after every commit. Commit = local save. Push = GitHub backup.
git log --oneline
Show full history of all save points
a3f2c1d feat: add chicken entrees (M1.1) 9b12e4a feat: initial repo structure (M0.2)
🍛 Your project's timeline. Each line = one commit.
git diff
Show exactly what changed line-by-line since last save
green lines = added red lines = removed
🍛 Review before committing so no surprises go up to GitHub

Rules
Present tense ("add", not "added") · lowercase everything · under 72 characters · milestone in brackets
Committing without running git status first
You might stage files you didn't intend to — including sensitive ones.
Always: git status → git add → git commit
Forgetting to push after committing
git commit saves locally only. Your laptop dies — the work is gone.
Commit and push are a pair. Always do both.
Committing node_modules/ or .env
.env exposes your secrets. node_modules is hundreds of MB of bloat.
Your .gitignore handles this. If you see them in git status, stop — check .gitignore.
Vague commit messages: "update", "changes", "fix"
Three months later you'll have no idea what "update" means.
Be specific: "feat: add Chicken Tikka Masala to menu-data.js (M1.1)"
Using full absolute paths when a relative path works
Typing ~/dev/joy-curry-tandoor/frontend/js when you're already in the project.
Inside the project: cd frontend/js is enough. Full paths for jumping from outside.
